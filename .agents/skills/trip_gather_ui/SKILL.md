# TripGather UI 개발 가이드라인

이 스킬은 TripGather 플랫폼 전반에 걸쳐 일관된 UI/UX를 유지하기 위한 가이드라인을 제공합니다. 이 표준을 준수함으로써 프리미엄하고 일관된 여행 테마 경험을 제공할 수 있습니다.

## 핵심 디자인 원칙

1.  **여행 테마**: 모든 요소는 여행의 설렘을 불러일으켜야 합니다 (티켓, 여권, 스탬프, 비행 경로 등).
2.  **프리미엄 글래스모피즘**: 모든 표면에는 `glass` 및 `glass-dark` 클래스를 사용합니다. 높은 `backdrop-filter: blur(12px)`을 유지합니다.
3.  **생생한 포인트 컬러**: 주요 액션에는 `--primary-orange`를 사용하고, 보조 포인트에는 `--secondary-purple` / `--secondary-blue`를 사용합니다.
4.  **모바일 우선 디자인**: 레이아웃은 최대 너비 `480px`에 최적화되어 있습니다.

## 표준 컴포넌트

### 1. 카드 컨테이너 (`Card.jsx`)
일반적인 콘텐츠 섹션에 사용합니다. 공통 글래스모피즘, 그림자 및 호버 애니메이션을 처리합니다.

- **Props**:
    - `glass`: 불리언 (기본값 `true`). 글래스모피즘 적용 여부.
    - `padding`: 불리언 (기본값 `true`). 표준 `24px` 패딩 적용 여부.
    - `animate`: 불리언 (기본값 `true`). 페이드인 애니메이션 적용 여부.

### 2. 티켓 레이아웃 (`TicketBase.jsx`)
바우처, 항공권 또는 미션 카드를 나타내는 모든 기능에 사용합니다.

- **구조**:
    - `header`: 상단 섹션용 React 엘리먼트 (절취선 위).
    - `footer`: 액션 영역용 React 엘리먼트 (하단 회색 배경).
    - `children`: 헤더와 푸터 사이의 메인 콘텐츠.
- **시각적 특징**: 점선 구분선 양 끝에 장식용 원형 컷아웃이 포함되어 있습니다.

### 3. 기본 버튼 (`PrimaryButton.jsx`)
모든 상호작용을 위한 표준 버튼입니다.

- **변형(Variants)**:
    - `primary`: 오렌지 그라데이션 (주요 액션).
    - `secondary`: 밝은 회색 (상세 보기와 같은 보조 액션).
    - `glass`: 투명한 흰색 테두리 (야간 모드 또는 이미지 오버레이).
    - `outline`: 테두리만 있는 형태 (낮은 우선순위 액션).

## 디자인 토큰 및 유틸리티

`index.css`에 정의된 토큰을 참조하세요:

- **색상**: `--primary-orange`, `--secondary-purple`, `--text-muted`.
- **아이콘**: 표준 크기 `16px` - `24px`의 `Lucide` 아이콘을 사용합니다.
- **레이아웃**:
    - `.flex-between`: 양끝 정렬.
    - `.gap-md`: 일관된 `16px` 간격.
    - `.label-muted`: 작고 굵으며 채도가 낮은 대문자 레이블.

## 컴포넌트 사용 가이드

| 기능 | 컴포넌트 |
| :--- | :--- |
| 사용자 프로필 정보 | `Card` |
| 여정 / 미션 | `TicketBase` |
| 네비게이션 아이템 | `.icon-circle` |
| 주요 액션 버튼 | `PrimaryButton (variant="primary")` |
