# TripGather Project Standards

This skill defines the global rules and communication standards for all contributors and AI agents working on the TripGather project.

## Communication Standard

1.  **Language**: All AI responses, explanations, summaries, and **AI-generated documents (Artifacts, Tasks, Plans, Walkthroughs)** MUST be provided in **Korean (한국어)**. This ensures consistency for the project owner.
2.  **Code Comments**: Code-level comments should remain in English for international developer compatibility, but documentation and high-level summaries should be in Korean.

## Project Structure Rules

- **Components**: New UI components must be placed in `frontend/src/components/UI/` if they are reusable, or organized by feature.
- **Resources**: Assets like icons and themes should follow the patterns established in `TripGatherUI` skill.
